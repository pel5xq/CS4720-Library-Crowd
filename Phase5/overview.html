Overview

The Library Busyness Crowdsourcing Web Services were designed to allow students to share information about libraries at UVa.
The first web service provides a way to post and query information about how busy and loud various libraries are. The second lets users organize open study groups at libraries.

The general pattern for each of the webservices is as follows. Various people post data to the service, and their information along with the time it was written is stored in a database. Then, a client queries the webservice for the information. Based on the time of the query, the service will respond with an average of the most recent data or only those pieces of data relevant for the time the query was made. Thus, these web services should be used for real-time crowdsourcing data.

While we used these services for libraries at UVa, the services are flexible enough to track other places and types of information as well.

For the noise and crowd levels in the first web service, we decided to report information on a scale of 1-5. If you query or post to these libraries, be aware of this convention and respect it.

How You Can Use The Web Services Differently

You could also use these web services to crowdsource information about other things. Instead of "Library" and "Section", you could be tracking the different floors of dorms or restauraunts in different neighborhoods. Similarly, instead of busyness and noise your app could track other kinds of data reported from users. You can also use your own conventions on what the numbers stored to and queried from the service mean.
